# MyExpenses Project Rules

1. Use natural, casual language instead of technical coding terms.
2. Explain things like talking to a friend who is not technical.
3. Use simple real-world analogies for complex logic or system behavior.
4. Keep explanations short, straight forward, and easy to picture mentally.
5. Focus on what is happening and why it is happening.
6. Use bullet points instead of long paragraphs whenever possible.
7. If technical terms are needed, explain them immediately in simple words.
8. Use conversational wording, phrasal verbs, and relaxed explanations.
9. Work only on the current branch, main.
10. Do not create, switch, commit, or push branches unless I clearly ask.
11. Do not use `/ultrareview` unless I explicitly request it.
12. Before changing files, inspect the related files and show the plan first.
13. Only apply changes after I clearly say: “apply approved changes”.
14. Do not delete, rename, overwrite, or move files without approval.
15. Keep changes focused only on the requested task.
16. Do not add unrelated improvements or remove existing features without approval.
17. Transactions is the source of truth for real spending.
18. Debts and TDG AVF / MONTH must reflect real Transactions values when showing actual spending.
19. Clearly separate:
* actual spending
* projected spending
* combined values
20. Use Brazilian currency formatting:
* R$ 1.532,26
* decimal comma
* thousands separator with dot
21. Keep the dashboard dark, premium, clean, and visually consistent.
22. Before finishing any task, always review:
* calculations
* logic flow
* component behavior
* UI consistency
* responsiveness
* charts and filters
* actual vs projected values
* broken connections between components
23. Validate totals, averages, percentages, debts, projections, and monthly calculations using the selected period and correct data source.
24. Review the system like checking parts of a machine:
* make sure everything still connects properly
* make sure one change did not quietly break another area
25. At the end of every task provide:
* validation summary
* possible risks
* future improvement ideas
* all modified and created files with locations
26. After the review, always suggest practical ideas for:
* UI polish
* smarter workflows
* automation
* premium dashboard improvements
* finance tracking
* better visualizations
* projection tools
* calculator improvements
* missing useful features
